Build, analyze, and deploy Web4 tooling

makeuniversal combines Makefile orchestration, C/C++ demos, installer scripts, and docs to help you prototype Web4 components.

Get started View on GitHub

Quickstart

Two fast ways to try the project: containerized builds or local/native builds.

Containerized (Docker)

make node
make python

These targets build container images for node and python development environments.

Native C demo

# Build lightweight server for testing
gcc -std=c11 -Wall -Wextra -O2 -o server/server server/server.c

# Run
./server/server

Playground C++ demos require project dependencies (klee/CRAXExpr) or building inside the dev container.

Highlights

  • Makefile & .mk modules - orchestration for container builds and docs.
  • C/C++ demos - ELF analyzer templates and small TCP gateway (proxy protocol parsing).
  • Installer scripts - Lmlm.dev contains a full installer for the Hermes-style environment.
  • Docs & whitepaper - architecture notes and a whitepaper included in the repo.
  • Pages CI - GitHub workflows for static docs are already present.

Contribute

Thanks for helping! A CONTRIBUTING.md (recommended) should describe how to run tests, linting and style. Quick pointers:

  • Create a branch per-change and open a Pull Request against main.
  • Run shellcheck on scripts and cppcheck/clang-tidy on C/C++ code.

Security: audit installer scripts prior to running on production systems.
